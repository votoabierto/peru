# VotoAbierto — Brand Guidelines

## Design Philosophy
Legibilidad primero. Objetividad siempre. Para todos los peruanos.

## Why light background?
Information platforms for public service use light backgrounds because:
- Higher legibility across all age groups and screen qualities
- Associated with newspapers, official documents, trust
- Dark themes feel like apps; light themes feel like information

## Typography
- **Noto Sans** — primary. Same family as international airport signage. Maximum legibility.
- **IBM Plex Mono** — data only. Vote counts, percentages, statistics.
- Rule: if a 65-year-old can't read it comfortably, it fails.

### Type scale
- Hero/Display: 48-64px, weight 700, color #111111
- H1: 28px, weight 700, color #111111
- H2: 22px, weight 600, color #111111
- H3: 17px, weight 600, color #222222
- Body: 16px, weight 400, line-height 1.7, color #333333
- Small/caption: 13px, weight 500, color #666666
- Labels/tags: 11px, weight 600, UPPERCASE, letter-spacing 0.08em, color #888888
- Stats/numbers: IBM Plex Mono, 14-18px, color #111111

## Color Principles
- **White first** — information before aesthetics
- **Institutional blue** (#1A56A0) — CTAs, links, active states. Non-partisan.
- **Peru red** (#D91023) — ONLY for national symbols. Never as UI accent. Strict.
- **Party colors** — NEVER used. All party names in gray (#777777).
- **Verdicts** — green/red/amber only for fact-check verdicts. Semantic only.

## Voice
- Objective, not opinionated
- Clear, not clever
- Peruvian, not political
- Concise — voters are busy

## What we never do
- Use party colors in UI
- Show polling/prediction ("who's winning")
- Use emotionally charged language
- Favor one candidate's framing over another's
